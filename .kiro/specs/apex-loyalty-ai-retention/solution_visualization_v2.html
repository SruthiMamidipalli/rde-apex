Agent Design

Four Specialized Agents

Each agent runs on the right model for its task. Agents have real autonomy — they can reject the pipeline or escalate beyond standard offers.

🔍
Signal Collector Agent
Claude 3 Haiku — fast, low-cost data gathering

Goal: Gather all available signals for a customer across all 6 systems in the minimum number of tool calls.


Tools: query_crm · query_shopify · query_yotpo · query_klaviyo · query_zendesk · query_analytics


Autonomy decisions:
• Decides the query order (most-predictive first)
• Skips systems marked stale (>6h cache)
• Re-queries if returned data looks anomalous
• Flags potential conflicts for the next agent


Output: Structured signal object passed to Conflict Detector

🔬
Conflict Detector Agent
Claude 3.5 Sonnet — complex semantic reasoning

Goal: Find contradictions between systems, identify leading vs lagging indicators, and assess whether the churn signal is genuine.


Reasons about:
• Signal conflicts (CRM 72/100 vs GA −60%)
• Temporal gaps (GA leads CRM by 2-3 weeks)
• Action gaps (14 campaigns sent, 8.2% open, no change)
• Seasonal vs structural patterns


Can REJECT pipeline: If the agent determines "not real churn, just seasonal" — it halts the pipeline, logs to audit, and no offer is generated. This prevents false-positive campaigns.


Output: Gap analysis + adjusted score → Offer Strategist

💡
Offer Strategist Agent
Claude 3.5 Sonnet — judgment and matching

Goal: Design the optimal retention intervention given the confirmed signals, customer tier, offer history, and margin headroom.


Tools: get_customer_tier · get_offer_history · get_offer_catalog · check_margin_headroom


Avoids: Repeating generic offers already ignored (e.g., two prior 10% vouchers rejected).


Can ESCALATE: "Standard offers won't work — needs personal DRI call." Routes to DRI queue instead of automated campaign.


Output: Offer details + confidence % → Engagement Crafter

✍️
Engagement Crafter Agent
Claude 3 Haiku — fast content generation

Goal: Produce ready-to-send outreach across email, SMS, and push — personalised to the specific signals, tone calibrated to risk level.


Tools: get_channel_preferences · get_brand_voice


Autonomy decisions:
• Chooses channel priority (CRITICAL = SMS first, not email)
• Names the specific support issue in the message
• Adapts tone: urgent for HIGH, warmer for MEDIUM


Feedback loop: If the offer description exceeds SMS char limit, it asks the Offer Strategist to simplify before drafting.


Output: 3-channel outreach drafts → Orchestrator final assembly

Monitor mode. Loyalty nudge via Klaviyo sequence. No agent invoked. Re-score in 7 days.
LOW  < 30
No action. Score logged. Scheduled recalc every 48 hours.

The Core Problem

System Conflicts — Why the CRM is Wrong

Five of six systems signal imminent churn. The CRM says the customer is healthy. Without a composite view, the wrong offer fires — or nothing fires at all.

Confirmed FactSourceConflicting SystemThe Gap
Sessions dropped 60% in 30 days Google Analytics Salesforce CRM CRM scores customer 72/100 healthy — uses purchase recency only, ignores behavioral intent
Zero redemptions in 8 weeks Yotpo Shopify Loyalty engine not tracking points-to-order conversion; generic 10% voucher sent
Unresolved ticket open 6 days Zendesk Yotpo + Klaviyo Loyalty engine and email platform unaware; reward email sent mid-open ticket
14 campaigns sent, 8.2% open rate Klaviyo Salesforce CRM CRM sees "campaign sent = action taken"; doesn't track disengagement. Campaigns continue unchanged.
AOV declined 34%; 3/5 orders used discount Shopify Yotpo Loyalty offer sent: generic 10% voucher — same as non-discount-dependent customers. Reinforces discount dependency.
